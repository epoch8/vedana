# Changelog

All notable changes to Vedana are documented here.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project aims to follow [Semantic Versioning](https://semver.org/spec/v2.0.0.html) once a first tagged release is cut.

Sections used per release:

- **Added** — new features.
- **Changed** — changes in existing functionality.
- **Deprecated** — soon-to-be-removed features.
- **Removed** — removed features.
- **Fixed** — bug fixes.
- **Security** — security fixes and improvements.

> Vedana is a monorepo. We version the project as a whole (single tag covering all packages). Per-package versioning may be introduced later if it becomes painful — until then, the version in this file is the version of the whole repository.
